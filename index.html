Metadata Sync

Java, Oracle, SQLite, JPA
We required a process for syncing the metadata (configuration) we had on the server with each device, so that when work was sent to the device, it could be displayed correctly. Prior to this project, each table in the metadata DB was being synced via XML and SOAP endpoints. This project included creating a method for generating a SQLite databased based off different tables in the production Oracle database and deploying it automatically to devices.
Challenges included understanding the subtle differences between Oracle / SQLite and creating test scenarios based on the large amounts of possible data that could be included.
We aimed to design the system in a way that would allow the process to be generic enough that adding new tables or columns to sync was a simple as adding some new SQL in a file. A few months later when the domain model simplfied this prooved to be a very helpful feature.
