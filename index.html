Metadata Sync

Java, Oracle, SQLite, JPA
We required a process for syncing the metadata (configuration) we had on the server with each users device in the field, so that when field work was sent out to the a users device, it could be displayed correctly. Prior to this project, each table in the metadata DB was being synced via XML and SOAP endpoints. This project included designing and implementing a method for generating a SQLite database, based off different tables in the production Oracle database and deploying it automatically to devices.
Challenges included understanding the subtle differences between Oracle / SQLite and creating test scenarios based on the large combinations of possible data that could be included.
We aimed to design the system in a way that would allow the process to be generic enough that adding new tables or columns to process was a simple as adding some new SQL in a file. A few months later when the domain model was modified, this proved to be a very helpful feature.

Stocktake application

Python, Flask, SQLite, XML
During my time as a McDonald's Restaurant manager, stocktakes were completed on paper (usually about 20 A4 sheets!). This process would take hours and was prone to data entry mistakes.
Sick and tired of completing these huge manual stocktakes I wrote a simple Python web application for completing the stocktake. Once a stocktake was complete, it would produce an XML file that was imported into the back-office PC.
The application was used across the 5 restaurants owned by the Owner/Operator of the Franchise I worked for. Parts of how it worked were used to develop an official stocktake website by McDonald's Head Office.
